1. Overview

In this lab, the VL53L1X time-of-flight (ToF) distance sensor(s) were integrated with the SparkFun RedBoard Artemis Nano via the Qwiic (I2C) bus. The main goals were: (1) verify the sensor I2C address using an I2C scan, (2) run two ToF sensors on the same bus by resolving the address conflict, (3) evaluate distance modes and basic performance (accuracy / repeatability / ranging time), and (4) log timestamped ToF + IMU data and transmit it over BLE for plotting.

2. Prelab

2.1 I2C Sensor Address

The VL53L1X ToF sensor communicates over I2C. Using an I2C scan on the Artemis, the sensor responds at 0x29 (7-bit address), which matches the default address.

Figure 1. I2C scan output showing the VL53L1X at 0x29.

2.2 Approach to Using Two ToF Sensors

Both ToF boards ship with the same default I2C address (0x29), so two sensors conflict on the same bus unless addressing is resolved. I considered two approaches:

  • Approach A (Used): XSHUT + change address once. Use each sensor’s XSHUT to enable only one device at boot time. Boot Sensor A alone, change its I2C address to e.g. 0x30, then enable Sensor B (keeps 0x29). After that, both sensors stay enabled and can be read in the same loop.
  • Approach B: XSHUT toggling every read. Keep both sensors at 0x29 and repeatedly toggle XSHUT to read one-by-one. This avoids changing the address but reduces effective sampling rate and complicates timing.

I used Approach A because it keeps both sensors enabled during runtime and is cleaner for later labs where sampling speed matters.

2.3 Placement & Missed-Obstacle Scenarios

Planned placement: mount two ToF sensors at the front (left/right), slightly yawed outward to cover a wider forward sector.

Likely miss cases include:

  • Obstacles outside the ToF field-of-view (narrow beam)
  • Thin poles/edges with small reflective area
  • Very dark/absorptive surfaces or strong specular reflections
  • Angled surfaces reflecting the laser away
  • Low obstacles blocked by chassis geometry

Qwiic/I2C uses 4 wires: 3.3V, GND, SDA, SCL. Artemis → Qwiic breakout → sensors in parallel. For the two-sensor approach (A), each sensor’s XSHUT is routed to a separate Artemis GPIO (extra wire per sensor, not on the 4-wire Qwiic plug).

3.3 ToF Distance Modes (Short / Medium / Long) and Chosen Mode

The sensor supports multiple distance modes: Short (~1.3 m), Medium (~3 m) (depending on library support), and Long (~4 m).

  • Short: better close-range performance / less long-range noise; good for indoor obstacle avoidance.
  • Long: larger range but more sensitive to reflectivity/angle and can be noisier.
  • Medium: compromise (library-dependent).

I used Short mode by calling distanceSensor.setDistanceModeShort();

3.6 ToF Speed: Non-blocking Loop + Limiting Factors

To avoid blocking while waiting for a measurement, the loop runs continuously and only reads a sensor when data is ready. The library supports checking readiness via checkForDataReady().

Limiting factors in practice: (1) sensor measurement timing budget / update period, and (2) I/O overhead (frequent Serial.print can dominate).

3.8 (Optional) 5000-level: IR Sensor Discussion

  • (1) IR distance sensor types (examples) + pros/cons:
    • IR triangulation (e.g., Sharp analog IR): uses the spot position on a detector to infer distance.
      Pros: simple, low cost, good for short range. Cons: non-linear response, sensitive to surface reflectivity/angle, limited range.
    • ToF (time-of-flight) (e.g., VL53L1X): measures the return time of emitted IR light.
      Pros: more direct geometry measurement, generally better linearity and repeatability. Cons: affected by low reflectivity (dark surfaces), specular/angled reflections, and strong ambient IR.
    • IR phase-shift ToF (common in some ToF cameras): measures phase difference of modulated IR.
      Pros: fast sampling, robust for mid-range. Cons: can suffer from multipath interference and requires more complex processing/calibration.
  • (2) Is the VL53L1X ToF sensitive to different colors/textures? Yes — because it relies on the strength/quality of returned IR light. In general:
    • Dark / matte surfaces absorb more IR → weaker return → noisier readings / shorter effective range.
    • Glossy / reflective surfaces can cause specular reflections (bounce away from sensor) → unstable or biased readings.
    • Rough textures often give more diffuse reflection → more stable return compared to shiny angled surfaces.
    • Large incidence angle (sensor not perpendicular to wall) reduces returned energy → measurement quality degrades.
